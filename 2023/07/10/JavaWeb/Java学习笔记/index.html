三、接口

声明接口

接口是对希望符合接口的类的一组需求，如果需要使用某方法，意味着类必须实现某接口。如果要使用Arrays类中的sort函数，则需要实现Comparable接口。

接口里的方法不提供实现，且自动声明为public。要声明一个类实现某接口，需要将类声明为实现给定的接口，且对接口中的方法提供定义。要声明一个类实现某接口，需要使用关键字implement：

接口的属性

接口不是类，不能使用new来声明一个接口，但是可以将一个接口变量与实现了这个接口的一个具体对象绑定。

接口也有继承体系，可以使用extends关键字拓展接口，使其从泛用性强拓展到专用性强：

接口中不能包含实例字段，但可以包含常量。在接口内的实例字段自动声明为public static final。

可以将接口方法显式标记为public，将实例字段标记为public static final，但java语言规范建议不要使用冗余的关键字。

散列集（HashSet）

有一种众所周知的数据结构，可以用来快速查找对象，被称作散列表。散列表为每个对象计算一个散列码。要想查找每个元素在表中的位置，要先计算其散列码，再通过运算找到桶的编号。如果桶中已经有了元素，这种现象被称作散列冲突（Hash Collision），这时需要将新对象与原对象进行比较，查看这个数据是否存在。

Java库提供了一个散列集，可以用add方法插入元素，使用contains方法查看元素是否存在。当访问元素时，会通过一种看起来随机的方式，只有不关心集合中元素顺序时才使用HashSet。
