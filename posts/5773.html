HTTP协议

请求数据格式

http请求的数据格式分为get和post请求，get请求的请求内容位于请求行内，有大小限制，但post位于请求体内，没有大小限制。

请求响应格式

请求响应的数据也分为post和get，这与上面一致。

  • 返回代码为以下形式：
代码 状态信息
1xx 响应中-临时响应码，表示请求已经接收，告诉客户端应该继续请求，如果完成，则忽略
2xx 成功
3xx 重定向-重定向到其他地方，让客户端再发起一次请求完成整个过程
4xx 客户端错误-处理发生错误，责任在客户端
5xx 服务器错误-处理发生错误，则仍在服务端
  • 常用的返回码：
状态码 英文描述 解释
200（！ OK 处理成功
302 Found 表示所请求的资源已经移动到另一个url，浏览器会自动访问这个页面
304 Not Modified 告诉客户端服务器的内容自上次请求后没变，直接用本地缓存即可
400 Bad Request 客户端有语法错误，服务器无法理解
404（！ Not Found 请求资源不存在，一般是url输入错误，或者网站资源被删除
403 Forbidden 没有权限
405 Method Not Allowed 请求方式有误，比如get误用了post
428 Precondition Required 服务器要求有条件的请求，必须携带特定的请求头
429 Too Many Requests 给服务器发送了太多的请求
431 Request Header Fields Too Large 请求头太大，服务器不愿意处理请求
500（！ Internal Server Error 服务器崩了
503 Service Unavailable 服务器尚未准备好，服务器刚刚启动，还未初始化很好
  • 常见的响应头：
字段 信息
Content-Type 表示响应内容的类型，例如text/html，application/json
Content-Length 表示该响应内容的长度（字节数）
Content-Encoding 表示响应压缩算法，比如gzip
Cache-Control 指示客户端如何缓存，例如max-age=300表示最多缓存300秒
Set-Cookie 告诉浏览器为当前页面所在的区域设置cookie

web服务器

web服务器对http协议操作进行封装，简化web开发。且能部署web项目，对外提供网上信息浏览服务。

  • Tomcat

这是一个轻量级web开源服务器，支持servlet，jsp等少量java ee规范。也被称为web容器，servlet容器。

Tomcat的基本使用

Tomcat是一个绿色软件，直接解压后，找到bin目录里的start.bat即可运行。若出现乱码，需要到logging配置文件里将UTF-8改为GDK。若启动后无法运行，需要配置环境变量JAVA_HOME至java安装路径。

Tomcat启动后，默认占用端口8080，可以在conf/server.xml中修改端口号。

将带有index.html的文件放在webapps中即可完成部署。

请求参数

请求简单参数

直接将参数名写在函数的参数列表即可。可以使用@RequestParam注解来指定请求的参数名，require标签指定了是否必须（默认为true）

请求实体参数

将对象名写在参数列表即可。

请求数组/集合参数

需要用@RequestParam指定参数为List<>

请求时间参数

时间参数将封装在一个LocalDate或LocalDateTime对象中，需要使用@RequestParam(patter=“”)来指定时间格式，注意请求的参数名必须和变量名一致，另外还需要注意日期标准格式是"yyyy-MM-dd HH:mm:ss"，M和H需要大写，否则传递参数不识别。

请求json参数

json参数需要封装在一个对象中，这个对象可以自定义。需要在对象前加上@RequestBody注解来声明这个一个json参数。

也可以指定多个参数。
